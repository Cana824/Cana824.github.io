Problem Solving Techniques

Problem Solving:

is a vital skill for overcoming challenges and an important ability for every developer.

Throughout the journey in Web Development, it is essential to build techniques to help you approach and tackle problems effectively.

There are various techniques such as:

  • Pseudocode
  • Trying something
  • Rubber Ducky method
  • Reading Error Messages
  • Console.logging
  • Googling
  • Asking Peers
  • Asking Coaches
  • Reflection

Pseudocode

Confidence: 6/10

Breaking code into written steps helps turn big problems into smaller, easier pieces. Writing everything out makes it easier to see the process, move steps around, or add and remove things when needed. Laying down the framework first is useful, but I often don't feel fully confident until I actually start writing bits of code.

Trying Something

Confidence: 9/10

Often the best way forward is just to try an idea and see what happens. I usually find that I know more than I thought once I give it a go. Trial and error is important, especially when problems first feel overwhelming.

Rubber ducky method

Confidence: 10/10

The rubber ducky method is my favourite. When I'm stuck, I explain the problem out loud-sometimes even just to myself. Talking it through helps organise my thoughts and often leads me to the solution.

Reading error messages

Confidence: 5/10

Error messages can be very useful for pointing me in the right direction, but they also confuse me quite often. They help highlight the part of the code that needs attention, but I don't always understand exactly what they mean.

Console.logging

Confidence: 8/10

Console.log helps me check what a variable is doing in my code. It's useful for seeing the current value or behaviour, but it still throws me off sometimes-especially when I get undefined or something unexpected.

Googling

Confidence: 8/10

Searching online with keywords helps me solve different parts of a problem. The tricky part is not always knowing what to search for if I don't know the thing exists. I usually try changing keywords or scrolling through examples until I find something that helps.

Asking peers for help

Confidence: 9/10

When I get stuck, I often ask my partner for help. I do my best to solve it first, but I also know it's better to ask before I get too frustrated. Peers are helpful because they've likely faced the same challenges during their learning.

Asking coaches for help

Confidence: 5/10

Asking coaches for help is harder for me. Even though I know they're there to support my learning, I usually try every other method before asking. Part of this comes from shyness, pride, or not wanting to sound silly. But if I really need to, I will ask.

Improving my process with reflection

Confidence: 7/10

After finishing code, I like to review it to make sure I really understand how everything fits together. Sometimes I need to rewrite sections multiple times before it works properly. Reflecting on my code helps reinforce what I've learned.

Blocked on a simple problem

What was the problem?

I got stuck in Super FlipFlop. I didn't understand how to use a function written elsewhere inside another function. The function only returned one answer from the array instead of the whole set.

What problem-solving techniques did you use?

I used Google, rubber ducking, and pseudocode. Talking through the code helped me realise I was missing a return. I also learned I needed to use let to store results in a new array. Later, I refactored and cleaned up extra code.

How did you feel throughout the process?

I felt confused and a bit frustrated, especially because I thought I had solved it but kept hitting the same issue. Once I realised my mistake, I felt relieved but also a little silly for forgetting the return.

What did you learn?

I learned the importance of reading the question carefully and making sure I return the correct value. I also learned that pseudocoding can help me organise my steps, even if it feels uncertain at first, and that refactoring can simplify my code.

Elegantly solved a problem

What was the problem?

I wrote a function to calculate the average of an array of numbers.

What problem-solving techniques did you use?

I used what I had already learned: declaring variables, looping, summing values, and dividing for the average. I didn't need to Google anything.

How did you feel throughout the process?

I felt confident and pleasantly surprised that I could solve it without outside help. It showed me that my practice was paying off.

What did you learn?

I learned that I am starting to memorise and apply common coding patterns. This gave me confidence that my learning is sticking.

Reluctant to ask for help

Reflect on a time you were reluctant to ask for help.

I was reluctant to ask for help when I was close to solving a piece of code on my own. I wanted the pride of finishing it solo.

What made you reluctant to do so?

I didn't want to admit defeat, so I kept pushing even when I was stuck for too long.

What might you try differently next time?

Next time, I'll ask for help sooner instead of being stubborn. This will save time, reduce frustration, and give me more space to reflect on the solution and learn from it.
